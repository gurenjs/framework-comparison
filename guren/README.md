# Minilog — Guren

The [SPEC.md](../SPEC.md) blog implemented on [Guren](https://github.com/gurenjs/guren),
scaffolded with `bunx create-guren-app` (`--auth`, SQLite) and
`bunx guren make:feature Post --fields "title:string,body:text" --policy`.

## Stack

- **Runtime:** Bun
- **Framework:** Guren — MVC controllers, session auth, policies, queued jobs
- **Database:** SQLite + Drizzle ORM (migrations via `bunx guren make:migration`)
- **Auth:** framework session auth (`this.auth`, `requireAuthenticated`, `requireGuest`)
- **Validation:** Zod schemas through `this.validateBody()` / `validateParams()` —
  422 responses and Inertia error bags are handled by the framework
- **Authorization:** `PostPolicy` enforced with `this.authorize()`
- **Frontend:** Inertia + React with typed page props and a typed `route()` helper
  generated by `bunx guren codegen`
- **Deferred work:** the welcome notification is a queued `Job` processed by an
  in-process queue `Worker`

## Setup

```bash
bun install
cp .env.example .env
bunx guren key:generate --write
bun run codegen    # generates .guren/ typed manifests
bun run db:migrate
bun run dev        # http://localhost:3333
```

## Tests

```bash
bun run test       # integration tests over app.fetch with a throwaway SQLite DB
```

## Verify like an agent would

```bash
bunx guren check   # routes ↔ controllers ↔ pages consistency
bunx guren audit   # validation/auth coverage on mutating routes
bun run typecheck
```
